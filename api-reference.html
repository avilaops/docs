Overview

This reference documentation covers all public APIs across the 120+ packages in the Arxis Core ecosystem. Each package follows consistent naming conventions and design patterns.

Documentation Format

Each API entry includes:

  • Signature: Function or type signature
  • Description: What it does and when to use it
  • Parameters: Input parameters and their types
  • Returns: Return type and value
  • Examples: Practical code examples
  • Errors: Possible error conditions

Core APIs

avx-std

Standard library extensions and common utilities.

Result<T, E>

Type alias for error handling across all Arxis packages.

pub type Result<T, E = Error> = std::result::Result<T, E>;

// Usage
use avx_std::Result;

fn process() -> Result<String> {
    Ok("success".to_string())
}

prelude

Common imports included in all Arxis packages.

use avx_std::prelude::*;

// Includes:
// - Result, Error
// - Arc, Mutex, RwLock
// - Duration, Instant
// - HashMap, HashSet
// - Vec, String

avx-error

Unified error handling system.

Error

Main error type used throughout Arxis Core.

pub enum Error {
    Io(std::io::Error),
    Parse(String),
    NotFound(String),
    Invalid(String),
    Timeout,
    // ... more variants
}

impl Error {
    pub fn kind(&self) -> ErrorKind { /* ... */ }
    pub fn message(&self) -> &str { /* ... */ }
}

Cryptography APIs

avx-crypto

Core cryptographic primitives.

hash::sha256(data: &[u8]) -> [u8; 32]

Computes SHA-256 hash of input data.

use avx_crypto::hash;

let data = b"Hello, World!";
let hash = hash::sha256(data);
println!("Hash: {:x?}", hash);

Ed25519Keypair

Ed25519 digital signature keypair.

use avx_crypto::Ed25519Keypair;

// Generate new keypair
let keypair = Ed25519Keypair::generate();

// Sign message
let message = b"Important message";
let signature = keypair.sign(message);

// Verify signature
assert!(keypair.verify(message, &signature));

Aes256Gcm

AES-256-GCM authenticated encryption.

use avx_crypto::Aes256Gcm;

let key = Aes256Gcm::generate_key();
let cipher = Aes256Gcm::new(&key);

// Encrypt
let plaintext = b"Secret data";
let ciphertext = cipher.encrypt(plaintext)?;

// Decrypt
let decrypted = cipher.decrypt(&ciphertext)?;
assert_eq!(plaintext, decrypted.as_slice());

Network APIs

avx-http

HTTP/2 client and server implementation.

Server::bind(addr: &str)

Creates a new HTTP server bound to the specified address.

use avx_http::{Server, Router, Response};

#[tokio::main]
async fn main() -> Result<()> {
    let app = Router::new()
        .get("/", |_| async { 
            Response::ok("Hello!") 
        })
        .post("/data", handle_data);

    Server::bind("0.0.0.0:8080")
        .serve(app)
        .await
}

Client::new()

Creates a new HTTP client with connection pooling.

use avx_http::Client;

let client = Client::new();
let response = client
    .get("https://api.example.com/data")
    .header("Authorization", "Bearer token")
    .send()
    .await?;

let body = response.text().await?;

Database APIs

avxdb

High-performance database engine.

Database::open(path: &Path)

Opens or creates a database at the specified path.

use avxdb::Database;

let db = Database::open("./data.db").await?;

// Insert data
db.insert("users", &user).await?;

// Query data
let users: Vec = db
    .query("SELECT * FROM users WHERE age > ?")
    .bind(18)
    .fetch_all()
    .await?;

AI & Machine Learning APIs

avx-tensor

N-dimensional array operations with GPU support.

Tensor::new(shape: &[usize])

Creates a new tensor with the specified shape.

use avx_tensor::Tensor;

// Create 2D tensor (matrix)
let a = Tensor::from_slice(&[1.0, 2.0, 3.0, 4.0], &[2, 2]);
let b = Tensor::from_slice(&[5.0, 6.0, 7.0, 8.0], &[2, 2]);

// Matrix multiplication
let c = a.matmul(&b);

// GPU acceleration
#[cfg(feature = "gpu")]
let gpu_tensor = a.to_gpu()?;

API Conventions

Naming

  • Types: PascalCase (e.g., HttpServer)
  • Functions: snake_case (e.g., connect_to_server)
  • Constants: SCREAMING_SNAKE_CASE (e.g., MAX_SIZE)
  • Modules: snake_case (e.g., network_utils)

Error Handling

All fallible operations return Result<T, Error>:

// Good
fn process() -> Result<Data> { /* ... */ }

// Avoid
fn process() -> Option<Data> { /* ... */ }

Async/Await

Async functions are marked with the async keyword:

async fn fetch_data() -> Result<Vec<u8>> {
    let response = client.get(url).send().await?;
    response.bytes().await
}

Builder Pattern

Complex types use the builder pattern:

let server = Server::builder()
    .bind("0.0.0.0:8080")
    .workers(4)
    .timeout(Duration::from_secs(30))
    .build()?;

Versioning

Arxis Core follows Semantic Versioning:

  • MAJOR: Breaking changes
  • MINOR: New features (backwards compatible)
  • PATCH: Bug fixes
